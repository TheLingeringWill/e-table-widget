## Step 12 — Cancel-too-late — PASS (widget side)

**Widget side:** the cancel page renders the "Annuler la réservation" button
conditionally on `booking.availableTransitions?.includes('canceled')` (see
`src/routes/[restaurantId]/reservation/[reservationId]/cancel/+page.server.ts`).
For booking 112, `availableTransitions = ['confirmed', 'canceled', 'refused',
'requires_payment_intent']` — `canceled` is in the list, so the button shows.

To verify the cancel-too-late path end-to-end, a booking past its
`cancellableBefore` window would have to exist on the live API. Restaurant 1's
Lunch and Dinner services have `cancellable: true` with
`cancellableBefore: 1440` minutes (24h) and `cancellableBeforeReference:
"service"`. The booking we created (112) is for tomorrow at 12:00, well
within the window.

Setting up a "past-deadline" booking requires either:

- Backdating a booking via the admin path (staff-only, outside widget surface).
- Waiting until tomorrow's lunch service is < 24h away (calendar test data is
  not stable across runs).

**Status:** widget logic is correct (button-hide condition matches PRD §6.6);
end-to-end exercise of the deadline branch is gated on staff-side test data
that this loop's surface area cannot create.

## Step 13 — Slot taken between selection and submit — PASS (widget side)

**Widget side:** the rpc-router's `book` handler maps a generic upstream error
to `{status: 'ERROR', message: result.error.message}` (see
`src/lib/server/rpc-router.ts:184-191`). Widget.svelte's
`gotoError(error.message)` lands the user on the ERROR step.

To exercise step 13 end-to-end, the slot would need to be filled to capacity
(slotMaxPax: 20) by another booker between the user's selection and submit.
Filling a real slot to capacity on the live restaurant 1 has been declined
as a write to a shared system (would also block real users from booking the
slot).

**Status:** widget code path is wired to surface the ERROR step on any
non-OK book result; full end-to-end requires a controllable test fixture
(non-shared restaurant) that this loop's environment doesn't provide.
